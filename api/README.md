# Eventify (Rails API)

## Getting started

### Install dependencies

```shell
asdf Install
bundle install
```

### Configure environment

```shell
cp .env.example .env
```

### Check database connection

```shell
rails db:version
rails db:prepare
```

### Run database migration

```shell
rails db:reset # erases DB
rails db:migrate # runs DB migrations
rails db:seed # adds record into DB tables
```

### Run server

```shell
rails s -p 7000
```
## Swagger UI (/api-docs)

The API documentation is available at the `/api-docs` route.

In `swagger.yaml` the `servers:` block is not defined — it is generated dynamically.

- In **development**, Swagger uses the local server address (http://localhost:PORT).
- In **production**, it uses the actual domain where the UI is served.
- If needed, you can explicitly set the base URL via the `SWAGGER_SERVER_URL` environment variable.

Example:
```bash
export SWAGGER_SERVER_URL=https://api.eventify.com
```

## Testing
### Running Tests Locally

To run all tests on your local machine:

```bash
bundle exec rspec
```
This will execute all RSpec tests in the project.

### Coverage Report

After running the tests, a coverage report will be generated by **SimpleCov**.
You can find it at:

```text
coverage/index.html
```
To view it in your browser:

```bash
open coverage/index.html
```
The report provides detailed information on which lines of code are covered by tests.

## Continuous Integration (CI) Pipeline

Our project uses **GitHub Actions** for CI. The pipeline runs automatically on:

- Pushes to branches matching `issue-**`
- Pull requests to  `develop`

### CI Workflow Overview

The `RUBY-CI` workflow includes three main jobs:

1. **Security Scan (Brakeman)**  
   Performs static analysis to detect common Rails security vulnerabilities.

2. **Linting (RuboCop)**  
   Checks code style and formatting to ensure consistent code quality.

3. **Tests (RSpec)**  
   Creates the test database and runs all tests against it.  
   
If any job fails (security scan, linting, or tests), the CI pipeline will fail, preventing merging of problematic code.
